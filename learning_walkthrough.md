## learning_walkthrough

Explain the application, feature, or flow so the user can deeply understand the codebase.

This is a learning walkthrough, not an audit.

---

### Goal

Help the user reach a level of understanding where they could confidently work within the code.

---

### Approach

Start with the core idea and purpose of the system, feature, or flow.

Explain the main concepts, abstractions, and responsibilities involved.

Show how those concepts are represented in the codebase and how the structure reflects the architecture.

Gradually move deeper:

- architecture
- system structure
- logic
- implementation details

When useful, bring in related parts of the system for context, but always tie them back to the main topic being explained.

Explain how data moves through the system and how control flows between components where it matters.

Highlight important assumptions, edge cases, and trade-offs made by the implementation.

---

### Depth

Assume the goal is deep understanding, not surface familiarity.

Go into lower-level details when needed to make behavior intuitive and predictable.

Prefer clarity and conceptual understanding over exhaustive enumeration of every line of code.

---

### Tone

Teach rather than document.

Use clear, natural explanations that help build mental models of the system.

Avoid rigid checklist-style explanations unless they genuinely improve clarity.
